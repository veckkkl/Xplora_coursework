## О проекте

**Xplora** — мобильное приложение под iOS для людей, которые хотят не просто помнить свои поездки, а **видеть их как историю освоения мира**.

Проект объединяет несколько сценариев в одном продукте:

- интерактивную карту мира;
- визуализацию посещённых стран и городов;
- travel journal / путевой дневник;
- wishlist мест, куда хочется поехать;
- персональную статистику путешествий.

Идея проекта - превратить путешествия в наглядный, эмоциональный и личный опыт: пользователь не просто хранит список поездок, а видит, **где он уже был, какие регионы открыл, и куда хочет отправиться дальше**. Это соответствует целям и функциональному описанию проекта в приложенных материалах.

## Зачем нужен Xplora

Большинство travel-приложений решают только одну задачу:

- либо показывают карту,
- либо дают список поездок,
- либо работают как заметочник,
- либо как wishlist.

**Xplora** задуман как единое пространство, где путешествия представлены сразу в нескольких измерениях:

- **визуально** — через карту и механику “тумана”;
- **лично** — через заметки, фото, адреса, даты и воспоминания;
- **прагматично** — через список желаемых мест и прогресс;
- **мотивационно** — через статистику исследования мира.

Проект ориентирован не только на “учёт поездок”, но и на чувство прогресса, исследования и личной истории путешествий. Это прямо соответствует аннотации и целям проекта.

## Ключевые принципы проекта

### 1. Нативный iOS-first подход
Проект разрабатывается как **нативное iOS-приложение** на Swift с использованием UIKit, а не как кроссплатформенный прототип. Выбранный стек зафиксирован в документации проекта.

### 2. Простая и расширяемая архитектура
Архитектура опирается на **Clean Architecture + MVVM**, чтобы:
- отделять UI от бизнес-логики;
- упрощать развитие функциональности;
- не смешивать работу с данными и presentation layer;
- поддерживать предсказуемую структуру для последующей упрощенной расширяемости.

### 3. Визуальность как часть продукта
Карта, туман, заметки, фото, прогресс по странам и континентам — это не “дополнительные функции”, а основа пользовательского опыта.

### 4. Local-first foundation
Xplora проектируется так, чтобы пользовательские данные о поездках, заметках и списках были доступны локально и не зависели от обязательного серверного backend на раннем этапе.

### 5. Итерационная разработка
Проект развивается поэтапно: сначала фундамент, потом карта, затем дневник, wishlist, статистика, профиль и полировка UX/UI. Это прямо отражено в плане этапов проекта.

## Архитектура

Проект ориентируется на сочетание **Clean Architecture + MVVM**, выбранное для отделения бизнес-логики, слоя данных и UI. Это зафиксировано в документации проекта.

### Слои

#### Presentation
Содержит:
- `ViewController`
- `ViewModel`
- роутинг и сборку UI-модулей
- визуальные компоненты

Назначение:
- отображение данных;
- обработка пользовательских событий;
- передача действий в `ViewModel`.

#### Domain
Содержит:
- сущности предметной области;
- repository-протоколы;
- use cases.

Назначение:
- бизнес-логика;
- инварианты и сценарии работы приложения;
- независимость от UIKit и конкретного способа хранения данных.

#### Data
Содержит:
- реализации репозиториев;
- storage / services;
- работу с источниками данных.

Назначение:
- сохранение и получение данных;
- инкапсуляция логики доступа к локальному хранилищу;
- интеграция с сервисами платформы.

#### Core
Содержит:
- общие инфраструктурные элементы;
- dependency injection;
- базовые технические компоненты приложения.

### Поток зависимостей

```text
ViewController
↓
ViewModel
↓
UseCase
↓
Repository (protocol)
↓
RepositoryImpl
↓
Storage / Platform Services
````

### Почему это важно

Такой подход позволяет:

* масштабировать функциональность без хаоса;
* не превращать `ViewController` в место для всей логики приложения;
* изолировать изменения;
* проще подключать Codex / AI-агентов к разработке без разрушения структуры.

## Getting Started

### Требования

Для локальной разработки нужны:

* macOS
* Xcode
* iOS Simulator или физическое iOS-устройство
* Swift
* Git

### Локальный запуск

1. Клонируй репозиторий:

```bash
git clone <repo-url>
cd Xplora
```

2. Открой проект в Xcode:

```bash
open Xplora.xcodeproj
```

3. Выбери target и simulator.

4. Запусти приложение через `Run` в Xcode.

### Что проверить после запуска

* открывается основной tab bar;
* карта загружается корректно;
* переходы между экранами не ломают navigation stack;
* экран заметки открывается и сохраняет данные предсказуемо;
* UI остаётся отзывчивым.

### Разрешения

Для части продуктового сценария приложению потребуется доступ к:

* геолокации;
* фото/камере — по мере развития photo flow.
